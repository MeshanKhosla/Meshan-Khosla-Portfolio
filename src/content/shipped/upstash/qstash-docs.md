QStash had several ways to get started: the REST API, TypeScript and Python SDKs, schedules, queues, callbacks, and signature verification. The information existed, but related material was spread across different parts of the documentation.

![The QStash Getting Started documentation card.](/shipped/qstash-docs.webp)

<p class="image-source">Image from <a href="https://upstash.com/docs/qstash/overall/getstarted" target="_blank" rel="noopener noreferrer">QStash documentation</a>.</p>

I reorganized the docs around those common starting points. I also separated the getting-started material, examples, SDK guides, and API reference so each section had a clearer purpose.
